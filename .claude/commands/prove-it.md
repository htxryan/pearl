# Prove It

If you've just completed a functional change, you must PROVE IT.

Ensure you have run the app end to end, and captured thorough screenshot evidence of the change working as expected, including edge cases.

Put your evidence screenshot and analysis in docs/proof/<beads-id>/.

All proof should be git-tracked, committed, and pushed. Then report the URL of each evidence screenshot to the user, so they can easily view them.
